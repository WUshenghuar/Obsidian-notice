# Spaced Repetition 复习序列设计指南

> 怎么规划复习节奏、怎么分牌组、怎么调参数才能最高效

---

## 一、先理解 Spaced Repetition 的工作原理

这个插件用的是 SM-2 算法（SuperMemo 的变体），核心逻辑很简单：

> 你每复习一次，系统根据你的掌握程度，自动计算这张卡片的下次出现时间。
> 
> 答对了 -> 下次间隔变长（比如 1天->3天->7天->16天...）
> 答错了 -> 强制明天再出现

所以你不需要自己排计划，你只需要每天打开插件，它会自动把该复习的卡片推给你。

---

## 二、设计你的牌组结构

### 方案一：当前状态（所有卡片在一个文件）

你现在所有 100+ 张卡片都放在 #flashcards 这一个文件里。好处是简单，坏处是复习时混在一起不分章节。

### 方案二：按章节分牌组（推荐）

用标签区分不同牌组，把闪卡文件里的 #flashcards 改成：

```
#flashcards/基础
#flashcards/生存期模型
#flashcards/需求管理
#flashcards/WBS
#flashcards/成本估算
#flashcards/进度管理
#flashcards/质量管理
#flashcards/风险管理
#flashcards/团队沟通
#flashcards/易错判断
```

然后在复习时，可以只复习某一个牌组：
Ctrl+P -> Spaced Repetition: Review by Tag -> 选 #flashcards/成本估算

### 方案三：按掌握程度分

- #flashcards/薄弱 -- 得分 < 60 的章节（每天必刷）
- #flashcards/稳定 -- 已经掌握的知识（偶尔刷）
- #flashcards/考前 -- 考前一周重点突击

> 建议先用方案一（现状）刷几天，等熟悉了再按方案二拆分

## 三、核心参数设置

打开 **设置 -> Spaced Repetition -> Scheduler**：

### 最重要的三个参数

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| New Cards per Day | 10~15 | 每天新学几张卡片。设太多会堆积如山 |
| Reviews per Day | 20~40 | 每天最多复习多少张。超过的顺延到明天 |
| Easy Interval | 4（天） | 点"容易"后卡片跳几天。设太长会漏复习 |

### 辅助参数

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| Base Ease | 250% | 默认值，每次答对间隔乘以 2.5 倍 |
| Lapses Interval | 0（分钟） | 答错后多久重新出现。设0=当天立即重测 |
| Maximum Interval | 90（天） | 最长间隔不超过90天，防止卡片永久沉睡 |

### 考前调参策略

```
考前 2 周：
  New Cards per Day -> 30  （集中学新卡）
  Reviews per Day   -> 80  （大量复习）
  Maximum Interval  -> 30  （缩短最长间隔，确保考前都出现）

考前 2 天：
  New Cards per Day -> 0   （不再学新东西）
  Reviews per Day   -> 999（不限量，把所有卡片过一遍）
```

## 四、每日复习流程设计

### 推荐节奏

```
每天早上（10分钟）：
  1. 打开 Obsidian
  2. Ctrl+P -> "Spaced Repetition: Review"
  3. 复习今天到期的卡片（10~20张）
  4. 顺手把新卡片加到对应笔记里

每周日（15分钟）：
  1. 打开插件统计面板看本周数据
  2. 看看哪些章节的卡片"忘记"率最高
  3. 针对性去翻对应章节的笔记重新理解
```

## 六、把做错的题也做成卡片

你现在已经有作业得分数据了，对得分低的章节，可以针对错题单独做卡片：

```
#flashcards/错题

关键路径决定项目的什么？::完成的最短时间

PERT期望值公式？::E = (O + 4M + P) / 6

ADM中箭线表示什么？::活动（任务）
```

## 七、进阶技巧

### 1. 埋线索卡片

不要只背答案，加一些帮你推导的线索：

```
怎么推导出成本平衡点公式？
?
让 自制总成本 = 购买总成本
自制 = 初始自制 + 月维护 x n
购买 = 初始购买 + 月维护 x n
解出 n = (自制初始 - 购买初始) / (购买月维护 - 自制月维护)
```

### 2. 用 Cloze 删词格式

对于公式类，可以用 {高亮删除} 格式：

```
功能点估算公式：FP = {UFC} x {TCF}
```

### 3. 结合知识图谱

打开 [[软件项目管理·知识图谱.canvas]]，看着图谱回忆各模块的关联，想不起来的就是需要加卡片的点。

## 八、一句话总结

> 你不用设计复习计划，插件替你算。你要做的只有三件事：
> 
> 1. 每天打开复习 10 分钟
> 2. 诚实评分（忘了就点"忘记"，别装会）
> 3. 每周看统计，发现薄弱章节，回去翻原笔记
