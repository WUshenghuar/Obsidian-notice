## 一、定额估算法（活动历时计算）

### 公式

```
T = Q / (R × S)
```

| 符号 | 含义 | 单位 |
|------|------|------|
| T | 活动历时 | 天/月 |
| Q | 任务工作量 | 人天/人月 |
| R | 人力数量 | 人 |
| S | 工作效率 | — |

### 例题

**课堂检测(1) Q21 / 课堂检测(2) Q25：**

> 定额估算法中，活动历时 T 的计算公式是？
> 
> A. T = Q + R + S
> B. T = Q × R × S
> C. T = (Q + R) / S
> D. T = Q / (R × S)
> 
> **答案：D**

## 二、PERT 计划评审技术

### 期望值公式

```
E = (O + 4M + P) / 6
```

| 符号 | 含义 |
|------|------|
| O | 乐观时间（Optimistic） |
| M | 最可能时间（Most likely） |
| P | 悲观时间（Pessimistic） |

### 标准差公式

```
δ = (P - O) / 6
```

### 例题

**课堂检测(3) Q17 — 期望值：**

> PERT 中活动历时的期望值计算公式是？
> 
> A. (O + 4M + P) / 6  ← ✅ 正确答案
> B. (O + M + P) / 3
> C. (O + M + 2P) / 4
> D. (O + 2M + P) / 4

**课堂检测(1) Q18 — 标准差：**

> PERT 中活动历时的标准差 δ 计算公式是？
> 
> A. δ = (P + O) / 6
> B. δ = (P + O) / 3
> C. δ = (P - O) / 3
> D. δ = (P - O) / 6  ← ✅ 正确答案

### 实战例题

> 某活动乐观时间 5 天，最可能时间 8 天，悲观时间 11 天：
> 
> 期望值 E = (5 + 4×8 + 11) / 6 = (5 + 32 + 11) / 6 = 48/6 = **8 天**
> 
> 标准差 δ = (11 - 5) / 6 = **1 天**

## 三、功能点估算（IFPUG）

### 公式

```
FP = UFC × TCF
```

| 符号 | 含义 |
|------|------|
| FP | 功能点（Function Point） |
| UFC | 未调整功能点计数 |
| TCF | 技术复杂度因子 |

### UFC 的五类功能项

| 缩写 | 全称 | 含义 |
|------|------|------|
| EI | 外部输入 | 用户输入数据 |
| EO | 外部输出 | 系统输出信息 |
| EQ | 外部查询 | 用户查询请求 |
| ILF | 内部逻辑文件 | 系统内部数据 |
| EIF | 外部接口文件 | 与其他系统交互的数据 |

### 例题

**课堂检测(1) Q1 — EIF 定义：**

> 外部接口文件（EIF）的定义是？
> 
> **答案：** 用户可以识别的一组逻辑相关数据，只能被引用，用于向另一个系统传送信息

**课堂检测(3) Q28 / Q33 — IFPUG 组成部分：**

> 以下哪个不是 IFPUG 功能点计数的功能项？
> 
> A. 内部接口文件（IIF） ← ✅ 正确答案（不在五类中）
> 
> 哪项不是 UFC 的组成部分？
> 
> B. 技术复杂度因子（TCF） ← ✅ 正确答案（TCF 是乘数，不是 UFC 的一部分）

### 易错提示

```
✅ 功能点（FP）与编程语言和技术无关
❌ 代码行（LOC）与编程语言有关
```

## 四、自制 vs 购买决策

### 核心公式

```
成本平衡点 n = (自制初始成本 - 购买初始成本) / (购买月维护费 - 自制月维护费)
```

**判断标准：**
- 项目生存期 < n → **购买** 更划算
- 项目生存期 > n → **自制** 更划算

### 例题 1

**课堂检测(2) Q33 / 课堂检测(3) Q22：**

> 某软件项目：
> - 自制方案：初始 25000 元，月维护 2500 元
> - 购买方案：初始 17000 元，月维护 2700 元
>
> 当软件生存期超过多少个月时，自制方案更划算？

**计算：**

```
n = (25000 - 17000) / (2700 - 2500)
  = 8000 / 200
  = 40 个月
```

> **答案：** 超过 40 个月时自制更划算

### 例题 2

**课堂检测(3) Q10（变体）：**

> 同上题数据，问：若项目生存期为 30 个月，应选哪种方案？

**分析：**
- 平衡点 n = 40 个月
- 30 < 40 → 购买方案成本更低

> **答案：** 选择购买方案 ✅

## 五、沟通渠道数

### 公式

```
沟通渠道数 = n × (n - 1) / 2
```

其中 n = 团队人数

### 例题

**作业20260611 第十章 Q4：**

> 当项目中有 20 个人时，沟通渠道最多有多少？

**计算：**

```
渠道数 = 20 × 19 / 2 = 190 条
```

> **答案：** 190

### 常见数值速查

| 人数 | 渠道数 |
|------|--------|
| 5 | 10 |
| 6 | 15 |
| 8 | 28 |
| 10 | 45 |
| 15 | 105 |
| 20 | 190 |

## 六、单位换算

### 公式

```
1 人月 = 22 人天
1 人天 = 8 有效工时
```

### 例题

**课堂检测(1) Q26：**

> 1 人月约等于多少人天？
> 
> A. 22 人天 ← ✅ 正确答案
> B. 176 人天（这是 22×8 的工时数）
> C. 10 人天
> D. 30 人天

## 七、其他常考概念型公式

### 成本平衡点（通用逻辑）

```
设定：自制总成本 = 购买总成本
设生存期为 n 个月：
  自制总成本 = 初始自制 + 月维护费自制 × n
  购买总成本 = 初始购买 + 月维护费购买 × n
  令两者相等，解出 n
```

### 用例点估算

```
Effort = UCP × PF
```

| 符号 | 含义 |
|------|------|
| UCP | 调整后的用例点 |
| PF | 生产率因子（工时/用例点） |
