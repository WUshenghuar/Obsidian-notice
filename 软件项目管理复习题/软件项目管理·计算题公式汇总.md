# 第一部分：成本估算（第六章）

## 一、定额估算法

### 公式

```
T = Q / (R × S)
```

| 符号 | 含义 | 单位 |
|------|------|------|
| T | 活动历时 | 天/月 |
| Q | 任务工作量 | 人天/人月 |
| R | 人力数量 | 人 |
| S | 工作效率 | — |

### 例题

**课堂检测(1) Q21：** 定额估算法中，活动历时 T 的计算公式是？→ **T = Q / (R × S)**

> **理解方法：** 把"工作量"想象成一块蛋糕，R×S 是"人×效率"即"吃蛋糕的速度"，历时=蛋糕量÷速度

## 二、PERT 三点估算 + Delphi 专家法

### PERT 核心公式

```
期望值 E = (O + 4M + P) / 6
标准差 δ = (P - O) / 6
方差     = δ²
```

| 符号 | 含义 |
|------|------|
| O | 乐观时间（Optimistic） |
| M | 最可能时间（Most likely） |
| P | 悲观时间（Pessimistic） |

> **理解方法：** 加权平均，最可能值权重是乐观和悲观的4倍，分母6=1+4+1

### 多个专家取平均

**作业20260416第六章 Q24（原题）：**

> 项目经理采用 Delphi 法，邀请3位专家估算：
> - 专家一：2万、7万、12万
> - 专家二：4万、6万、8万
> - 专家三：2万、6万、10万
>
> 求项目成本估算值。

**解题步骤：**

```
第1步：每位专家用PERT算期望值
  专家一: E1 = (2 + 4×7 + 12) / 6 = 42/6 = 7万
  专家二: E2 = (4 + 4×6 + 8) / 6 = 36/6 = 6万
  专家三: E3 = (2 + 4×6 + 10) / 6 = 36/6 = 6万

第2步：取三位专家的平均值
  项目估算 = (7 + 6 + 6) / 3 ≈ 6.3万
```

> **理解方法：** 先每个人用PERT算出自己的"加权平均"，再把人平均。两步平均而已。

## 三、PERT 概率计算

### 正态分布概率

```
±1σ  →  68.3%    （1个标准差范围内）
±2σ  →  95.4%
±3σ  →  99.7%
```

**作业20260514第七章 Q27（原题）：**

> 乐观6天，悲观24天，最可能12天。求历时介于10天到16天的概率？

**解题步骤：**

```
第1步：算期望值和标准差
  E = (6 + 4×12 + 24) / 6 = 78/6 = 13天
  δ = (24 - 6) / 6 = 3天

第2步：看10天和16天在哪个区间
  10 = 13 - 3 = E - δ
  16 = 13 + 3 = E + δ
  所以是 [E-δ, E+δ] = ±1σ 区间

第3步：查概率 → 68.3%
```

> **理解方法：** 算出期望和标准差后，看目标区间是±几倍标准差，直接套68.3%/95.4%/99.7%

## 四、功能点估算（IFPUG）

### 公式

```
FP = UFC × TCF
```

### UFC 的五类功能项

| 缩写 | 含义 |
|------|------|
| EI | 外部输入（External Input） |
| EO | 外部输出（External Output） |
| EQ | 外部查询（External Query） |
| ILF | 内部逻辑文件（Internal Logical File） |
| EIF | 外部接口文件（External Interface File） |

### 事务功能点复杂度

> 由 **FTR**（文件类型引用数）和 **DET**（数据元素类型）共同决定

**作业20260416第六章 Q6：**

> 事务类型功能点复杂度级别主要由____和____决定 → **FTR, DET**

> **理解方法：** 记口诀"FTR是文件数，DET是数据项数"，两个维度交叉查表得复杂度

### NESMA 三种估算模式

**作业20260416第六章 Q18：**

> NESMA 功能点估算的三种模式：**指示功能点计数、估算功能点计数、详细功能点计数**
> 
> ❌ COSMIC 方法不是 NESMA 的模式

## 五、COCOMO 模型

### 中等 COCOMO 公式

```
Effort = a × (KLOC)^b × EAF
```

| 项目类型 | a | b |
|---------|---|---|
| 有机型 | 2.4 | 1.05 |
| 半嵌入型 | **3.0** | **1.12** |
| 嵌入型 | 3.6 | 1.20 |

### 例题

**作业20260416第六章 Q25（原题）：**

> 预计 50KLOC 代码量，半嵌入型，中等 COCOMO 模型，可靠性为很高级别（取值1.3），其他属性正常。
> 
> （1）计算项目人月规模？（2）2万元/人月，总费用？

**解题步骤：**

```
第1步：代入公式
  Effort = 3.0 × 50^1.12 × 1.3

第2步：计算
  50^1.12 ≈ 79.9
  Effort ≈ 3.0 × 79.9 × 1.3 ≈ 311.8 人月

第3步：算费用
  总费用 = 311.8 × 2 = 623.6 万元
```

> **理解方法：** COCOMO 就是一个"乘乘乘"公式：基数 × (代码量^指数) × 调整因子。考试会给 a、b 值，直接代入计算器按就行。

## 六、IBM 模型

### 公式

| 要算什么 | 公式 |
|---------|------|
| 代码行数 L | L = 功能点 × 每功能点代码行（C语言≈150LOC/FP） |
| 工作量 E | E = 5.2 × L^0.91 |
| 项目时间 D | D = 4.1 × L^0.36 |
| 人员 S | S = 0.54 × E^0.6 |
| 文档 DOC | DOC = 49 × L^1.01 |

### 例题

**作业20260416第六章 Q26（原题）：**

> 85个功能点，C语言，估算代码行数、工作量、时间、人员、文档。

**解题步骤：**

```
第1步：算代码行
  L = 85 × 150 = 12,750 行 = 12.75 KLOC

第2步：算工作量
  E = 5.2 × 12.75^0.91 ≈ 52.7 人月

第3步：算项目时间
  D = 4.1 × 12.75^0.36 ≈ 10.2 月

第4步：算人员
  S = 0.54 × 52.7^0.6 ≈ 5.8 人

第5步：算文档
  DOC = 49 × 12.75^1.01 ≈ 641 页
```

> **理解方法：** 一步一步来，先算代码行，后边的公式都是已知数代入。考试会给指数值，用计算器按即可。

## 七、自制 vs 购买决策

### 公式

```
成本平衡点 n = (C自制初始 - C购买初始) / (C购买月维护 - C自制月维护)
```

- 生存期 < n → **购买**
- 生存期 > n → **自制**

### 例题（课堂检测多次出现）

> 自制：初始25000元，月维护2500元
> 购买：初始17000元，月维护2700元
> 
> n = (25000-17000) / (2700-2500) = 8000/200 = **40个月**

> **理解方法：** 设两者总成本相等解出n。分子是初始成本的差额，分母是月维护费的差额。超过n个月就"省回来的钱超过了多掏的初始投资"。

## 八、单位换算

```
1 人月 = 176 有效工时（22×8）
```

## 九、网络图时间参数

### 六个核心参数

| 缩写 | 含义 | 计算公式 |
|------|------|---------|
| ES | 最早开始 | 顺推法 |
| EF | 最早结束 | EF = ES + 历时 |
| LS | 最迟开始 | 逆推法 |
| LF | 最迟结束 | LF = LS + 历时 |
| TF | 总浮动 | TF = LS - ES = LF - EF |
| FF | 自由浮动 | 不影响紧后活动的前提下可延迟的时间 |

### 记忆方法

```
顺推法：从前往后，取最大值（多个前置活动中，取最晚的那个EF）
逆推法：从后往前，取最小值（多个后置活动中，取最早的那个LS）
总浮动 = 最迟 - 最早（开始或结束都行）
```

### 易错题

**作业20260514第七章 Q17：**

> 下面说法中不正确的是？
> 
> A. EF = ES + duration   ✓
> B. LS = LF - duration   ✓
> C. TF = LS - ES = LF - EF  ✓
> **D. EF = ES + lag  ← ❌ 正确答案（正确是 ES + duration，lag 是滞后量）**

## 十、活动依赖关系

### 四种依赖类型

| 类型 | 含义 | 示例 |
|------|------|------|
| FS（结束-开始） | A做完才能开始B | 编码→测试 |
| SS（开始-开始） | A开始B才能开始 | 设计→文档 |
| FF（结束-结束） | A结束B才能结束 | 测试→验收 |
| SF（开始-结束） | A开始B才能结束 | 很少用 |

### 三种依赖性质

| 性质 | 含义 |
|------|------|
| **强制性**（硬逻辑） | 物理上必须这样，如"编码完成才能测试" |
| **软逻辑**（组织关系） | 人为决定的顺序 |
| **外部依赖** | 项目外部的依赖 |

**作业20260514第七章 Q18：**

> "软件编码完成之后，我才可以对它进行软件测试" → **强制性依赖关系**

## 十四、决策树分析

### 公式

```
EMV（期望货币值）= Σ（概率 × 收益/损失）
```

判断标准：**选择 EMV 较大的方案**

### 例题 1：简单决策

**作业20260618第十一二章 Q31（原题）：**

> 采用方案A，掌握新工具概率30%（获利5万），不能掌握概率70%（损失1万）。
> 是否该采用方案A？

**解题：**

```
EMV = 30% × 5 + 70% × (-1)
    = 1.5 - 0.7
    = 0.8 万 > 0
→ 应该采用方案A
```

### 例题 2：方案对比

**作业20260618第十一二章 Q32（原题）：**

> 甲乙两种方案，各有不同状态和概率，收益表如下：

```
方案甲：EMV = 75
方案乙：EMV = 93  ← 乙更大

→ 选择乙方案
```

> **理解方法：** 每个分支=概率×收益，所有分支加起来就是期望值。哪个方案期望值高就选哪个，别忘了收益是正的、损失是负的。

## 十五、合同类型计算

### CPFF（成本加固定费用）

```
合同金额 = 实际成本 + 固定费用
```

**作业20260618第十一二章 Q8（原题）：**

> CPFF合同，估计成本10万，固定费用1.5万，当成本提高到20万时，合同金额？

```
合同金额 = 20 + 1.5 = 21.5 万
```

> **理解方法：** 无论成本多少，固定费用不变。成本涨了，甲方全担，乙方稳拿1.5万。

### CPIF（成本加激励费用）

```
总价 = 实际成本 + 费用 + 奖励/惩罚
```

**作业20260618第十一二章 Q28（原题）：**

> 预计成本20万，利润4万，奖励分配80/20（甲方80%/乙方20%），实际成本降至16万，求总价？

**解题：**

```
第1步：算节省的钱
  节省 = 20 - 16 = 4万

第2步：乙方分得的奖励
  乙方奖励 = 4 × 20% = 0.8万

第3步：总价
  总价 = 实际成本16 + 利润4 + 奖励0.8 = 20.8万
```

> **理解方法：** 提前完成省下来的钱，按比例分。甲方拿大头（80%），乙方拿小头（20%）作为激励。

### 合同类型风险对比

> **卖方（乙方）风险最大：固定总价**（成本超支自己扛）
> **买方（甲方）风险最大：成本加固定费用**（成本超支甲方担）

**作业20260618第十一二章 Q27：**

> 卖方承担风险最大的合同类型？→ **固定总价（FFP）**
